## Deployment Options

<Tabs>

    - ✅ Instant setup in under 5 minutes
    - ✅ Automatic updates and security patches
    - ✅ Built-in backups and disaster recovery
    - ✅ 99.9% uptime SLA
    - ✅ No infrastructure management required
    
    **Get Started**: [app.pager.team](https://app.pager.team)
  </Tab>
  
  <Tab title="Self-Hosted">

    - ✅ No data leaves your environment

  </Tab>
</Tabs>

## Self-Hosted Deployment

Deploy Pager on your own AWS infrastructure for complete control.

### System Requirements

<Tabs>
  <Tab title="AWS Services">
    **Required AWS Services**:
    - **Lambda**: Serverless compute for backend functions
    - **API Gateway**: REST API management
    - **RDS**: PostgreSQL database (or compatible)
    - **S3**: File storage and static assets
    - **CloudFormation**: Infrastructure as code
    - **Secrets Manager**: Secure configuration storage
    - **CloudWatch**: Logging and monitoring
  </Tab>
  
  <Tab title="Development Environment">
    **Local Development Requirements**:
    - **Node.js**: 18.17.0 or higher
    - **AWS CLI**: Configured with appropriate permissions
    - **SAM CLI**: For serverless deployment
    - **Docker**: For local testing and builds
    - **Git**: Version control
  </Tab>
  
  <Tab title="Third-Party Services">
    **External Dependencies**:
    - **Supabase**: Database and real-time subscriptions
    - **OpenAI**: AI agent functionality
    - **Google OAuth**: Authentication (optional)
  </Tab>
</Tabs>

### Pre-Deployment Setup

<Steps>
  <Step title="AWS Configuration">
    Set up your AWS environment:
    
    <CodeGroup>
    ```bash AWS CLI Configuration
    # Configure AWS CLI with your credentials
    aws configure
    
    # Verify access to required services
    aws sts get-caller-identity
    aws lambda list-functions --max-items 1
    ```
    </CodeGroup>
  </Step>
  
  <Step title="Clone Repository">
    Get the Pager source code:
    
    <CodeGroup>
    ```bash Clone Repositories
    # Clone frontend repository
    git clone https://github.com/your-org/pager-frontend.git
    cd pager-frontend
    
    # Clone backend repository  
    git clone https://github.com/your-org/pager-backend.git
    cd pager-backend
    ```
    </CodeGroup>
  </Step>
  
  <Step title="Install Dependencies">
    Install required packages:
    
    <CodeGroup>
    ```bash Frontend Dependencies
    cd pager-frontend
    npm install
    
    # Install development tools
    npm install -g @vercel/cli
    ```
    
    ```bash Backend Dependencies
    cd pager-backend
    npm install
    
    # Install SAM CLI (macOS with Homebrew)
    brew install aws-sam-cli
    
    # Or using npm
    npm install -g @aws-sam/cli
    ```
    </CodeGroup>
  </Step>
</Steps>

### Backend Deployment

Deploy the serverless backend to AWS:

<Steps>
  <Step title="Configure Parameters">
    Set up your deployment configuration:
    
    <CodeGroup>
    ```json config/parameters.json
    {
      "openai_api_key": "sk-your-openai-api-key",
      "supabase_url": "https://your-project.supabase.co",
      "supabase_service_role_key": "your-service-role-key",
      "supabase_anon_key": "your-anon-key",
      "google_client_id": "your-google-oauth-client-id",
      "google_client_secret": "your-google-oauth-secret",
      "jwt_secret": "your-secure-jwt-secret",
      "environment": "production"
    }
    ```
    </CodeGroup>
    
    <Warning>
      Store sensitive values in AWS Secrets Manager for production deployments
    </Warning>
  </Step>
  
  <Step title="Build and Deploy">
    Deploy the backend services:
    
    <CodeGroup>
    ```bash Deploy Backend
    # Build and deploy using SAM
    sam build
    sam deploy --guided
    
    # Follow the prompts to configure:
    # - Stack name (e.g., pager-backend-prod)
    # - AWS region (e.g., us-east-1)
    # - Confirm changes and create IAM roles
    ```
    </CodeGroup>
  </Step>
  
  <Step title="Note API Gateway URL">
    After deployment, save the API Gateway URL from the outputs:
    
    <CodeGroup>
    ```bash Get Outputs
    aws cloudformation describe-stacks \
      --stack-name pager-backend-prod \
      --query 'Stacks[0].Outputs'
    ```
    </CodeGroup>
  </Step>
</Steps>

### Frontend Deployment

Deploy the Next.js frontend application:

<Steps>
  <Step title="Configure Environment">
    Set up environment variables:
    
    <CodeGroup>
    ```bash .env.production
    NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
    NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
    SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
    OPENAI_API_KEY=sk-your-openai-api-key
    NEXT_PUBLIC_API_GATEWAY_URL=https://your-api-gateway.amazonaws.com/prod
    NEXT_PUBLIC_APP_URL=https://your-domain.com
    ```
    </CodeGroup>
  </Step>
  
  <Step title="Deploy to Vercel">
    Deploy using Vercel (recommended):
    
    <CodeGroup>
    ```bash Deploy to Vercel
    # Install Vercel CLI if not already installed
    npm install -g vercel
    
    # Deploy to Vercel
    vercel --prod
    
    # Configure environment variables in Vercel dashboard
    # or using CLI:
    vercel env add NEXT_PUBLIC_SUPABASE_URL production
    vercel env add NEXT_PUBLIC_SUPABASE_ANON_KEY production
    # ... add all environment variables
    ```
    </CodeGroup>
  </Step>
  
  <Step title="Alternative: AWS Deployment">
    Deploy to AWS using Serverless Next.js:
    
    <CodeGroup>
    ```bash AWS Frontend Deploy
    # Build for production
    npm run build
    
    # Deploy to S3 + CloudFront
    npx @serverless/aws
    
    # Or use AWS Amplify
    amplify init
    amplify add hosting
    amplify publish
    ```
    </CodeGroup>
  </Step>
</Steps>

### Database Setup

Configure your PostgreSQL database:

<Steps>
  <Step title="Supabase Setup">
    Create a new Supabase project:
    
    1. Visit [supabase.com](https://supabase.com) and create a new project
    2. Note your project URL and API keys
    3. Configure authentication providers (Google OAuth, email/password)
    4. Set up row-level security policies
  </Step>
  
  <Step title="Database Schema">
    Run the database migrations:
    
    <CodeGroup>
    ```sql Initial Schema
    -- Run these SQL commands in your Supabase SQL editor
    -- or use the migration files in supabase/migrations/
    
    -- Enable required extensions
    CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
    CREATE EXTENSION IF NOT EXISTS "vector";
    
    -- Create initial tables
    -- (Schema files available in the repository)
    ```
    </CodeGroup>
  </Step>
  
  <Step title="Real-time Configuration">
    Enable real-time subscriptions:
    
    <CodeGroup>
    ```sql Real-time Setup
    -- Enable realtime for required tables
    ALTER PUBLICATION supabase_realtime ADD TABLE messages;
    ALTER PUBLICATION supabase_realtime ADD TABLE channels;
    ALTER PUBLICATION supabase_realtime ADD TABLE workspaces;
    ```
    </CodeGroup>
  </Step>
</Steps>

## Production Configuration

### Security Hardening

<AccordionGroup>
  <Accordion title="Environment Security">
    - **Use AWS Secrets Manager** for all sensitive configuration
    - **Enable VPC** for Lambda functions if required
    - **Configure WAF** on API Gateway for DDoS protection
    - **Set up CloudTrail** for audit logging
    - **Enable GuardDuty** for threat detection
  </Accordion>
  
  <Accordion title="Database Security">
    - **Enable RLS policies** in Supabase
    - **Use connection pooling** for Lambda database connections
    - **Configure backup retention** policies
    - **Set up monitoring** and alerting
    - **Enable SSL/TLS** for all database connections
  </Accordion>
  
  <Accordion title="Application Security">
    - **Configure CORS** policies appropriately  
    - **Set up rate limiting** on API Gateway
    - **Enable request logging** for debugging
    - **Use JWT tokens** with appropriate expiration
    - **Implement input validation** on all endpoints
  </Accordion>
</AccordionGroup>

### Monitoring and Observability

<Steps>
  <Step title="CloudWatch Setup">
    Configure comprehensive logging and monitoring:
    
    <CodeGroup>
    ```yaml CloudWatch Configuration
    # Add to SAM template.yaml
    Globals:
      Function:
        Environment:
          Variables:
            LOG_LEVEL: INFO
        Tracing: Active
        
    # Enable X-Ray tracing
    Resources:
      MyFunction:
        Type: AWS::Serverless::Function
        Properties:
          Tracing: Active
    ```
    </CodeGroup>
  </Step>
  
  <Step title="Custom Metrics">
    Set up business metrics tracking:
    - Message volume and delivery rates
    - User engagement and active sessions
    - Webhook delivery success rates
    - AI agent usage and response times
  </Step>
  
  <Step title="Alerting">
    Configure alerts for critical issues:
    - High error rates or function failures
    - Database connection issues
    - Unusual traffic patterns
    - Security-related events
  </Step>
</Steps>
